# LAB_0 - Проектирование REST API для социальной сети

## Цель

Спроектировать API без реализации бэкенда: описать endpoints, входные/выходные данные и ошибки в формате OpenAPI.

## Что реализовано

В файле `openapi.yaml` описаны 5 операций:
1. Добавление в друзья.
2. Удаление из друзей.
3. Получение списка друзей.
4. Получение профиля пользователя.
5. Публикация поста.

## Основные сущности

- `UserProfile`: профиль пользователя.
- `Friend`: элемент списка друзей.
- `Post`: запись в ленте.
- `Error`: единый формат ошибки (`code`, `message`).

## Что важно для сдачи

1. Пояснить, почему для чтения используется `GET`, для создания `POST`, для удаления `DELETE`.
2. Показать, что в спецификации есть успешные ответы и ошибки (`400`, `404`, `409`).
3. Объяснить, что API описан в OpenAPI 3.0 и готов для Swagger-инструментов.

## Как проверить

- Открыть `openapi.yaml` в Swagger Editor.
- Проверить, что все 5 операций видны и имеют схемы запросов/ответов.
